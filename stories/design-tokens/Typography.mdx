# Typography Tokens

Typography tokens define the font families, sizes, weights, and other text-related properties used throughout the design system.

## Font Families

The design system uses system font stacks for optimal performance and native feel across different platforms.

## Usage Examples

### Headings

```tsx
<h1 className="font-sans text-4xl font-bold">
  Page Title
</h1>

<h2 className="font-sans text-3xl font-semibold">
  Section Heading
</h2>

<h3 className="font-sans text-2xl font-semibold">
  Subsection Heading
</h3>
```

### Body Text

```tsx
<p className="font-sans text-base font-normal">
  Regular body text for paragraphs and content.
</p>

<p className="font-sans text-sm text-muted-foreground">
  Secondary or helper text.
</p>
```

### Code

```tsx
<code className="font-mono text-sm">
  const example = "code snippet";
</code>

<pre className="font-mono text-xs">
  function hello() {
    return "world";
  }
</pre>
```

### Labels and UI Text

```tsx
<label className="font-sans text-sm font-medium">
  Form Label
</label>

<span className="font-sans text-xs text-muted-foreground">
  Helper text
</span>
```

## CSS Variables

```css
/* Font families */
.element {
  font-family: var(--font-sans);
  font-family: var(--font-serif);
  font-family: var(--font-mono);
}

/* Using Tailwind classes is recommended */
.heading {
  @apply font-sans text-2xl font-bold;
}
```

## Best Practices

1. **Use semantic HTML**: Use appropriate heading tags (`h1`-`h6`) for proper document structure
2. **Maintain hierarchy**: Use consistent font sizes and weights to establish clear visual hierarchy
3. **Limit font weights**: Stick to 2-3 font weights per project for consistency
4. **Consider readability**: Use appropriate line heights and letter spacing for body text
5. **Test across devices**: Ensure typography scales well on different screen sizes
